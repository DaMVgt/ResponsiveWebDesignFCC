Introduction

This is the introduction to the Github documentation page. People use GitHub to build some of the most advanced technologies in the world. Whether you’re visualizing data or building a new game, there’s a whole community and set of tools on GitHub that can help you do it even better.

How to Install Github

  • Step 1: Navigate to the download page of official git website and download Git client for windows.
  • Step 2: By running the installer file, the wizard will progress with the usual following step
    • Welcome screen
    • License agreement
    • Installation folder
    • Choose the components to be installed.
    • Start menu configuration
    along with these, wizards proceeds further with configuration specifics, which will be explained in the consequent steps.
  • Step 3: Adding the PATH environment

    By default, the installer will choose ‘Use Github from Github Bash Only’.
  • Step 4: Configure the terminal emulator to use with Git Bash.

Setting Global Git Username and Email

To set your global commit name and email address run the git config command with the --global option.

git config --global user.name "Your Name" git config --global user.email "youremail@yourdomain.com"

Github Commands

Suppose developer needs to prepare one new repository in his local environment. For initializing the same they have to use init commands, which helps them for initializing GitHub project repository locally.

Git init

Suppose developer needs to create one specific repository of Github in their local copy from the specific remote location. Then they have to execute clone command for copying the same remote repository in the local environment in specific location.

Git clone ssh://something.com/[username]/[repository_name].git

This GitHub command is mainly used for identifying GIT created status in local repository. It provides proper information between a syncing status of local and Github repository to the end user.

Git status -s

Used for adding one specific file in staging area.

Git add [file_name.doc]

Used for adding all files

Git add .

Commit all the required changes.

Git commits –m [“message for commit”]

Helping for mapping local repository’s origin branch with remote repository through SSH.

Git remote add origin ssh://something.com/[user_name]/[repository_name].git

It does a similar task like above, pushing entire local repository branch to the remote repository, extra utility of using this command is also remembered for this specific branch for future reference.

Git push –u origin [branch_name]

Helping for pulling entire changes from the remote repository to local.

Git pull origin [branch_name]

Displaying all the log information of one specific project, which can provide every information with committed changes.

Git log
